### 3.2 完整题目类型定义

```typescript
type QuestionType = 'choice' | 'fill' | 'input' | 'matchstick' | 'shape-match'

type Difficulty = 1 | 2 | 3 | 4 | 5

interface Exercise {
  id: string                      // 唯一ID
  type: QuestionType             // 题型
  difficulty: Difficulty          // 难度 1-5
  difficultyLabel: string        // 难度标签
  stem: string                    // 题干
  image?: string                   // 图像资源ID
  options?: string[]               // 选项（选择题必填）
  answer: string                  // 标准答案
  acceptableAnswers?: string[]    // 可接受答案
  solution: string[]              // 解析步骤
  hints: string[]                 // 提示
  commonMistakes: CommonMistake[] // 常见错误
  source: QuestionSource          // 来源
  knowledgePoints: string[]       // 关联知识点
  estimatedTime: number          // 预计用时（秒）
  irtParams: { a: number; b: number; c: number } // IRT参数
  tags: string[]                 // 标签
}

interface CommonMistake {
  mistake: string
  reason: string
  correction: string
  errorLayer: 'L1' | 'L2' | 'L3' | 'L4'
}

interface QuestionSource {
  type: 'textbook' | 'competition' | 'exam' | 'original'
  name: string
  year?: number
  original?: string
}
```

### 3.3 难度分级标准

| 难度 | 含义 | IRT b 值范围 |
|------|------|----------------|
| 1 ★ | 入门，直接套用公式 | -1.0 ~ -0.5 |
| 2 ★★ | 基础，需要理解概念 | -0.5 ~ 0.0 |
| 3 ★★★ | 进阶，3~5步，有陷阱 | 0.0 ~ 0.6 |
| 4 ★★★★ | 拔高，5+步，综合应用 | 0.6 ~ 1.2 |
| 5 ★★★★★ | 竞赛，需巧思/模型 | 1.2 ~ 2.0 |

## 四、课后测试抽题算法

### 4.1 抽题流程

```
用户点击 "开始课后测试"
    │
    ▼
[1] 获取当前知识点 topicId
    │
    ▼
[2] 读取题库文件 question-bank/g{年级}/{topicId}.json
    │
    ▼
[3] 读取用户能力数据
    │   ability = 用户对该知识点的 IRT 能力估计
    │
    ▼
[4] 智能选题
    │   ├─ 按难度区间过滤
    │   ├─ 按 IRT 概率选最适合题
    │   ├─ 保证题型多样性
    │   └─ 保证难度覆盖
    │
    ▼
[5] 生成测试 session (5~8 道题)
    │
    ▼
[6] 答题后即时判断对错 → 更新用户能力
    │
    ▼
[7] 测试结束: 正确率 + 错题 + 建议
```

## 五、题目编写指南

### 5.1 编写流程

1. 打开对应的 Markdown 讲解文件，理解核心概念
2. 阅读例题卡/变式卡，理解典型解法
3. 按 5 档难度编写题目
4. 每题必填 stem/answer/solution/hints/commonMistakes/source
5. 保存到 `public/content/question-bank/g{年级}/{topicId}.json`

### 5.2 难度梯度示例（以和倍问题）

| 难度 | 题目难度 | 考察能力 |
|------|---------|--------|
| 1 ★ | 两数之和为10，大数是小数的4倍，求小数 | 直接套用公式 |
| 2 ★★ | 苹果梨共20个，苹果是梨的3倍。苹果几个？ | 情境转化 |
| 3 ★★★ | 甲是乙的2倍，丙是甲的3倍，三数之和为81 | 建立统一段数 |
| 4 ★★★★ | 从上层拿15本到下层后上层是下层2倍 | 倍数变化 + 倒推 |
| 5 ★★★★★ | 甲分得的是乙丙之和的2/3... | 比例转化为和倍模型 |

### 5.3 真题来源建议

| 题目类型 | 来源 | 对应难度 |
|----------|------|---------|
| 基础题 | 人教版各年级课本习题改编 | 1-2 ★ |
| 基础应用题 | 人教版苏教版北师大版教材改编 | 2-3 ★★-★★★ |
| 典型奥数题 | "华罗庚金杯" / "希望杯" 初赛题改编 | 3-4 ★★★-★★★★ |
| 中档竞赛题 | "走美杯" / "数学大王" 改编 | 4 ★★★★ |
| 高级竞赛题 | 华杯赛复赛题 / 迎春杯题改编 | 4-5 ★★★★-★★★★★ |
| 小升初 | 历年民办初中入学测试题 | 5 ★★★★★ |

## 八、风险与应对

| 风险 | 影响 | 概率 | 应对 |
|------|------|------|------|
| 题库量太大 | 上线延迟 | 中 | 先完成基础 600 题覆盖，后续分批补齐 |
| 题目答案写错 | 用户流失 | 中 | 双人审查 + 每题自我检查 + 真实答题数据识别 |
| JSON 手写易错 | 解析失败 | 低 | 提供 JSON 模板 + 验证脚本 |
| 高年级难度控制不准 | 孩子受挫 | 中 | 先保守出题（偏易），真实数据动态调整 |
| 题库文件太多加载慢 | UI卡顿 | 低 | 懒加载按专题 |
