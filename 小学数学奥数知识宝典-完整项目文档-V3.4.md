## 一、产品概述

### 1.1 产品定位

一个网页可访问、手机可安装（APK）、完全离线可用的数学学习平台。

**核心价值：** 100% 覆盖 1-6 年级人教版课内知识点 + 奥数七大模块（计算、应用题、几何、数论、计数、数字谜、组合逻辑）全部内容。每个知识点配备六种学习卡片、虚拟教具、关联游戏、自适应练习，遵循"五步深度教学法"确保学习效果。

### 1.2 目标用户

| 用户 | 场景 | 核心需求 |
|------|------|---------|
| 学生（1-6年级） | 系统学数学、打基础、拓展思维 | 有趣的、能坚持的学习方式 |
| 家长 | 辅导孩子，但不懂怎么教 | 傻瓜式辅导脚本、孩子学情透明 |
| 教师 | 备课参考、课堂教具、分层练习 | 体系化内容、可视化教具 |

### 1.3 产品形态

| 形态 | 说明 | 使用场景 |
|------|------|---------|
| 网页版 | 部署到 GitHub Pages / 服务器 | 电脑浏览器访问，PWA 可离线 |
| APK版 | Android WebView 包装 | 手机安装，完全离线使用 |
| PWA | 网页版支持添加到主屏幕 | 类 App 体验，无需安装 |

### 1.4 核心特点

- **完全离线**：所有内容、游戏、教具、练习数据均存储在本地
- **双端运行**：一套代码，网页 + Android APK 双端部署
- **深度教学**：每个知识点遵循"五步深度教学法"，从情境唤醒到反思建构完整闭环
- **游戏化学习**：12+ 款数学游戏，在玩中巩固知识
- **虚拟教具**：6 大交互式教具，把抽象概念可视化
- **智能练习**：基于 IRT 的自适应刷题 + SM-2 间隔复习 + 错因分类靶向干预
- **激励系统**：宠物养成 + 成就徽章 + 经验等级 + 主题皮肤 + 里程碑仪式
- **家长赋能**：三层支持系统（傻瓜模式/理解模式/拓展模式）+ 辅导仪表盘

## 二、技术架构

### 2.1 技术栈

| 层次 | 技术 | 版本 | 说明 |
|------|------|------|------|
| 前端框架 | Vue 3 | 3.4+ | Composition API + `<script setup>` + TypeScript |
| 构建工具 | Vite | 5.x | 快速构建，HMR |
| 路由 | Vue Router | 4.x | Hash 模式（本地文件兼容） |
| 状态管理 | Pinia | 2.x | 按模块拆分 Store |
| 数学公式 | KaTeX | 0.16+ | 轻量，离线渲染 |
| 图表 | ECharts | 5.x | 按需引入，雷达图/柱状图/热力图 |
| Markdown | marked | 12.x | 自定义渲染器支持 Vue 组件 |
| 本地搜索 | FlexSearch | 0.7+ | 纯前端全文搜索，多类型分组 |
| 数据存储 | localStorage + IndexedDB | — | idb 库封装 |
| PWA | vite-plugin-pwa | 0.19+ | Service Worker 离线支持 |
| APK | Android WebView | — | 极简壳，仅加载本地文件 |

### 2.2 架构原则

- **单文件 ≤ 300 行**：`.vue` 文件模板+脚本+样式合计不超 300 行，超过则拆分子组件或提取逻辑到 `.ts` 文件
- **纯函数逻辑分离**：游戏算法、IRT 引擎、SM-2 算法等纯逻辑独立为 `.ts` 文件，便于测试
- **内容与逻辑分离**：Markdown 文件负责知识内容，Vue 组件负责交互逻辑
- **功能模块独立**：每个功能模块（游戏、教具、学习）自包含，可单独开发、测试、替换
- **离线优先**：所有核心功能不依赖网络，网络功能为可选增强

### 2.3 存储方案

| 数据类型 | 存储方式 | 说明 |
|---------|---------|------|
| 用户设置 | localStorage | 主题、字体、声音等 |
| 学习进度 | IndexedDB | 知识点掌握度、学习路径节点状态 |
| 练习记录 | IndexedDB | 答题历史、用时、正确率、错误层级 |
| 错题本 | IndexedDB | 错题 + 错误类型 + 错误层级 + 复习记录 |
| 成就数据 | IndexedDB | 已解锁成就、经验值、等级 |
| 宠物状态 | IndexedDB | 成长阶段、情绪、装饰、日记 |
| 笔记/书签 | IndexedDB | 用户笔记和高亮内容 |
| 草稿数据 | IndexedDB | 多页草稿自动保存 |
| 诊断数据 | IndexedDB | 诊断结果、能力雷达图历史 |
| 亲子辅导记录 | IndexedDB | 辅导时长、模式、评分 |

### 4.1 行数红线

- 组件模板超过 100 行 → 拆分子组件
- 脚本逻辑超过 150 行 → 提取到 composables 或专属 `.ts` 文件
- 样式超过 80 行 → 提取到独立 CSS 文件
- Store 超过 100 行 → 拆分业务逻辑到 services

### 4.3 命名规范

| 类型 | 规范 | 示例 |
|------|------|------|

| 函数/变量 | camelCase | loadContent, masteryLevel |
| 常量 | UPPER_SNAKE_CASE | MAX_PRACTICE_COUNT |
| TypeScript 接口 | 描述性名称 | UserProfile, ExerciseData |

### 4.5 状态管理规范

- **全局 Store（stores/）**：仅放跨模块共享的数据（用户信息、设置）
- **功能 Store（在 features/ 内）**：各功能模块的状态，独立管理
- 禁止 Store 之间直接引用，通过事件或 composables 通信
- 复杂业务逻辑放在 services/ 中，Store 只调用服务方法

### 4.6 类型安全要求

## 六、内容编写规范

### 6.1 "五步深度教学法"卡片编排规范

每个知识点（★级以上）必须包含以下五个步骤对应的卡片，按顺序排列：

| 步骤 | 卡片类型 | 核心问题 | 学生行为 | 最少内容要求 |
|------|---------|---------|---------|-------------|
| 第一步 | 情境故事卡 | "我为什么要学这个？" | 阅读/观看生活场景 | 1个贴近生活的场景描述 |
| 第二步 | 概念构建卡 | "这个知识到底是什么？" | 拖拽/观察/自己发现 | 1个交互教具 + 概念定义 |
| 第三步 | 例题精讲卡 | "我该怎么解决？" | 跟分步动画，每步自己先试 | 1道母题 + 完整分步解析 |
| 第四步 | 变式挑战卡 | "换样子我还会吗？" | 独立解决变式题 | ≥3道变式（覆盖≥4个变式维度） |
| 第五步 | 亲子共学卡 | "我能讲出来吗？" | 口述过程/出题/教家长 | 1份对话脚本 + 1道自主题 |

### 6.2 Markdown 文件结构

每个知识点一个 .md 文件，使用自定义容器标记不同类型的卡片：

```markdown
# 和倍问题

## 📌 它是什么？
和倍问题是已知两个数的和以及它们的倍数关系，求这两个数各是多少的问题。

## 🎯 为什么重要？
它是小学应用题的第一道分水岭，后续差倍、盈亏、鸡兔同笼都以此为基础。

## 📋 需要什么基础？
- 理解"倍"的含义（8 是 4 的 2 倍）
- 会画简单的线段图
- 能解决两步应用题

---

::: story-card
### 小明和小红的苹果
小明和小红一起摘苹果，两人一共摘了 30 个。小明摘的个数是小红的 4 倍。
"我摘得比你多！"小明得意地说。小红不服气："那是因为你摘得快！"
你能帮他们算一算，每人摘了多少个吗？
:::

::: concept-card
### 什么是"1份量"？
在两个数量进行比较时，把较小的那个量看作"1份"（也叫"1倍数"），
另一个量是它的几倍就是"几份"。

<SegmentDiagram :segments="[1,4]" total="30" />

**关键发现**：只要用**总和 ÷ 总份数**，就能算出 1 份是多少！

::: parent-notes
### 给家长的话
"1份量"是和倍问题的核心概念。如果孩子理解困难，
可以用积木或糖果实物操作：一堆小的（1份），一堆大的（4份），
让孩子自己数一数大的是小的几倍，总共是几份。
:::
:::

::: example-card
### 母题：和倍问题基本型
**题目**：小明和小红一共有 30 个苹果，小明的苹果数是小红的 4 倍。两人各有多少个？

**第一步**：找"1份量"——小红更少，小红是 1 份量
> 为什么？因为"小明的个数是小红的4倍"，被比的那个是1份量

**第二步**：画线段图——小红画 1 段，小明画 4 段

**第三步**：数总份数——1 + 4 = 5（份）

**第四步**：求 1 份量——30 ÷ 5 = 6（个）→ 小红

**第五步**：求其他量——6 × 4 = 24（个）→ 小明

**第六步**：验算——6 + 24 = 30 ✓，24 ÷ 6 = 4 ✓

**核心公式**：$1份量 = 和 \div (倍数 + 1)$

::: expand-thinking
### 拓展思考
如果总数变成 45 个，倍数变成 2 倍，答案是多少？
如果倍数不变，总数变成了 60 个，答案又是多少？
你发现了什么规律？
:::
:::

::: variant-card
### 变式一：暗和问题
一个长方形的周长是 60 厘米，长是宽的 4 倍。长和宽各是多少？
:::

::: variant-card
### 变式二：三个量的和倍
甲、乙、丙三个数的和是 120，甲是乙的 2 倍，丙是乙的 3 倍。三个数各是多少？
:::

::: variant-card
### 变式三：非整数倍
小明和小红一共有 35 个苹果，小明的苹果数比小红的 4 倍还多 5 个。两人各有多少个？
:::

::: mistake-card
### 常见错误 1：直接用和除以倍数
**错误**：30 ÷ 4 = 7.5，认为小红有 7.5 个
**原因**：忘记把倍数 + 1，总数对应的是（倍数 + 1）份，不是倍数份
**纠正**：画线段图，数清楚一共有几份
**错误层级**：L2（方法错误）
:::

::: mistake-card
### 常见错误 2：找错 1 份量
**错误**：把"小明的苹果数是小红的 4 倍"理解为小明是 1 份
**原因**：谁更少谁才是 1 份量，"A 是 B 的几倍"中 B 是 1 份量
**纠正**：口诀——"是"字后面是 1 份量
**错误层级**：L2（概念错误）
:::

::: parent-child-card
### 亲子共学脚本（傻瓜模式）
**家长**："你来帮我分析一下这道题。谁是 1 份量？"
**孩子**：（可能说小明或小红）
**家长**（如果孩子说错）："你看，小明的个数是小红的 4 倍，谁更少？"
**孩子**："小红是 1 份！"
**家长**："太好了！那你画图，小红画 1 段，小明画几段？"
**孩子**："4 段！"
**家长**："数一数，总共几份？"
**孩子**："5 份！"
**家长**："5 份对应 30 个，1 份是多少？"（给孩子至少 30 秒思考，不要催）
**孩子**："30 ÷ 5 = 6 个！"
**家长**："太棒了！所以小红 6 个，小明呢？"
**孩子**："6 × 4 = 24 个！"
**家长**："自己验算一下。6 + 24 = 30，24 ÷ 6 = 4，完全正确！你是怎么想到的？"
```

每道母题的解题步骤必须按以下四层结构编写：


## 七、教学法核心设计

### 7.1 三阶思维引导策略

针对不同难度的知识点，采用不同的引导深度：

| 难度 | 引导策略 | 提示次数 | 示例（以鸡兔同笼为例） |
|------|---------|---------|------------------------|
| ★-★★ | 示范型：先完整演示一遍，再让孩子模仿 | 无限制 | 先播放"假设全是鸡"的完整动画，再让孩子做变式 |
| ★★★ | 脚手架型：分步提示，每步一个引导问题 | 每步1次提示 | "第一步：如果我们假设笼子里全是鸡，总脚数是多少？" |
| ★★★★ | 苏格拉底型：只给最模糊的提示，让孩子自己探索 | 总共1次提示 | "你想想，鸡和兔最大的区别是什么？" |
| ★★★★★ | 挑战型：不给任何提示，完成后有特殊成就 | 0次提示 | 完全独立解决，获得"独立思考者"徽章 |

### 7.2 错因四层分类与靶向干预

| 错误层级 | 错误类型 | 具体表现 | 系统干预策略 |
|---------|---------|---------|-------------|
| L1：输入层 | 审题错误 | 漏看条件、看错数字、不理解题意 | 推送"读题三步法"训练：圈关键词→画图翻译→复述题意 |
| L2：加工层 | 概念错误 | 对核心概念理解有误（如找错1份量） | 回退到概念构建卡，重新操作教具 |
| L2：加工层 | 方法错误 | 解题思路偏差（如用错公式、步骤遗漏） | 展示标准步骤与用户步骤的并排对比 |
| L3：执行层 | 计算错误 | 进退位遗漏、口诀记错、小数点错位 | 推送对应计算类型的每日特训 |
| L4：元认知层 | 思维定势 | 只会套模板，题目变形就不会 | 推送"反套路"变式题，刻意训练条件识别 |

### 7.3 完整学习闭环（7步）

```
预评估 → 情境唤醒 → 概念建构 → 方法探索 → 变式迁移 → 反思建构 → 后测验证
   ↑                                                              ↓
   ←—————————— 未通过：回退到薄弱环节 ————————————————
   ←—————————— 通过：标记掌握，进入间隔复习队列 ———————————
```

- **预评估**：学习新知识点前，系统快速检测前置知识是否牢固（3-5道题）。前置知识掌握度<80%则先推送前置复习。
- **后测验证**：学完知识点后，随机抽取 5 道题（覆盖基本型+变式型），正确率 ≥ 80% 且用时 ≤ 标准时间 1.5 倍才标记为"已掌握"。

### 7.4 学习路径分流机制

```
预评估
├── 前置知识掌握度 ≥ 80% → 正常学习新知识
├── 前置知识掌握度 60-79% → 快速复习前置知识（10分钟微课）
└── 前置知识掌握度 < 60% → 必须先重新学习前置知识

后测验证
├── 正确率 ≥ 80% → 标记掌握，进入间隔复习
├── 正确率 50-79% → 针对性补充（哪类错题回哪张卡片）
└── 正确率 < 50% → 重新学习整个知识点，降低初始难度
```

### 7.5 间隔复习关联网络

在 SM-2 算法基础上增加知识点关联复习：

- 复习知识点 A 时，自动检查 A 的前驱知识点和易混知识点的掌握度
- 如果前驱知识点的掌握度下降（如从 5 星降到 3 星），优先复习前驱知识
- 如果易混知识点（如"和倍"与"差倍"）近期错误增多，推送对比练习

## 十一、构建与部署

### 11.1 Vite 配置

```typescript
// vite.config.ts
import { defineConfig } from 'vite'
import vue from '@vitejs/plugin-vue'
import { VitePWA } from 'vite-plugin-pwa'

export default defineConfig({
  plugins: [
    vue(),
    VitePWA({
      registerType: 'autoUpdate',
      workbox: {
        globPatterns: ['**/*.{js,css,html,md,json,svg,png,woff2}']
      }
    })
  ],
  resolve: {
    alias: { '@': '/src' }
  },
  build: {
    rollupOptions: {
      output: {
        manualChunks: {
          'vendor-vue': ['vue', 'vue-router', 'pinia'],
          'vendor-math': ['marked', 'katex'],
          'vendor-charts': ['echarts'],
        }
      }
    }
  }
})
```

### 11.2 一键构建脚本

```bash
#!/bin/bash
# scripts/build-all.sh

echo "=== 1/4 安装依赖 ==="
npm install

echo "=== 2/4 构建 Web 版本 ==="
npm run build

echo "=== 3/4 复制到 Android 工程 ==="
rm -rf android/app/src/main/assets/www
cp -r dist/ android/app/src/main/assets/www/

echo "=== 4/4 构建 APK ==="
cd android && ./gradlew assembleRelease && cd ..

echo ""
echo "=== 构建完成 ==="
echo "Web 版本: dist/"
echo "APK 文件: android/app/build/outputs/apk/release/app-release.apk"
ls -lh android/app/build/outputs/apk/release/app-release.apk
```

### 11.3 代码体积控制策略

| 策略 | 说明 |
|------|------|
| KaTeX 字体子集 | 仅包含数学符号 + 拉丁字母常用字符 |
| ECharts 按需引入 | 仅注册雷达图、柱状图、热力图 |
| Three.js 动态导入 | 仅在立体几何观察器中按需加载 |
| 游戏 Canvas 2D 优先 | 避免 WebGL 开销 |
| 图片 SVG 优先 | 矢量格式，体积小 |
| 路由懒加载 | 所有功能模块路由使用 () => import() |
| manualChunks 分包 | 基础库、数学库、图表库独立 chunk |
